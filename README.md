# ระบบลงคะแนนดาว-เดือน

ระบบลงคะแนนแบบ 2 รอบ (รอบที่ 1 ผู้เข้าประกวดหญิง G01-G10, รอบที่ 2 ผู้เข้าประกวดชาย M01-M12)
พร้อมจอแสดงผลกราฟแท่งแบบเรียลไทม์สำหรับให้คนภายนอกดู

## หน้าจอที่มีในระบบ

| หน้า | URL | ใช้ทำอะไร |
|---|---|---|
| จอแสดงผลสาธารณะ | `http://<host>:3000/display.html` | ขึ้นโปรเจกเตอร์/จอทีวี ให้คนดูกราฟแท่งสด อัปเดตทุก 2 วินาที |
| หน้ากรอกคะแนน | `http://<host>:3000/input.html` | ให้ทีมงานกรอกคะแนน กด "🌸 +1 ดอกไม้ (+30 คะแนน)" ทีละครั้งต่อผู้เข้าประกวด |
| หน้าแอดมิน | `http://<host>:3000/admin.html` | สลับรอบที่ 1/2 ที่จะแสดงบนจอสาธารณะ และรีเซ็ตคะแนน (ต้องใส่รหัสผ่าน) |

> เปลี่ยน `<host>` เป็น IP ของเครื่องที่รัน Docker เช่น `http://192.168.1.50:3000/display.html`
> ถ้ารันบนเครื่องเดียวกับที่เปิดดู ใช้ `localhost` ได้เลย

## วิธีรันด้วย Docker

ต้องมี Docker และ Docker Compose ติดตั้งไว้แล้ว

```bash
# อยู่ในโฟลเดอร์โปรเจกต์นี้
docker compose up --build -d
```

ระบบจะรันที่พอร์ต **3000** และคะแนนจะถูกเก็บไว้ในโฟลเดอร์ `./data`
บนเครื่อง host (ผ่าน volume) ทำให้ **restart container แล้วคะแนนไม่หาย**

ปิดระบบ:

```bash
docker compose down
```

ดู log:

```bash
docker compose logs -f
```

## ตั้งรหัสผ่านแอดมิน

แก้ค่า `ADMIN_PASSWORD` ใน `docker-compose.yml` ก่อนรันจริง (ค่าเริ่มต้นคือ `changeme123`)

```yaml
environment:
  - ADMIN_PASSWORD=รหัสผ่านที่ตั้งเอง
```

จากนั้น `docker compose up --build -d` ใหม่อีกครั้ง

## กติกาการให้คะแนน

- กรอกคะแนนแบบ "1 ดอกไม้ = 30 คะแนน" ทุกครั้งที่กดปุ่มในหน้ากรอกคะแนน
- คะแนนของผู้ชายและผู้หญิงแยกกันคนละชุดข้อมูล ไม่ปนกัน
- จอแสดงผลสาธารณะจะโชว์เฉพาะกลุ่มที่ตรงกับ "รอบ" ที่ตั้งไว้ในหน้าแอดมิน
  (รอบที่ 1 = หญิง, รอบที่ 2 = ชาย) แม้จะกรอกคะแนนของอีกฝั่งไว้ล่วงหน้าได้ตลอดเวลา
  แต่จะไม่โชว์บนจอจนกว่าแอดมินจะสลับรอบ

## โครงสร้างไฟล์

```
starmoon-vote/
├── Dockerfile
├── docker-compose.yml
├── server/
│   ├── package.json
│   └── server.js          # Express API + เก็บคะแนนเป็นไฟล์ JSON
├── public/
│   ├── display.html        # จอแสดงผลสาธารณะ (กราฟแท่ง)
│   ├── input.html           # หน้ากรอกคะแนน
│   └── admin.html           # หน้าแอดมิน (สลับรอบ/รีเซ็ต)
└── data/                    # (สร้างอัตโนมัติ) เก็บคะแนนแบบ persistent
```

## ปรับแต่งเพิ่มเติมที่ทำได้

- เปลี่ยนชื่อผู้เข้าประกวดจากรหัส (G01, M01 ฯลฯ) เป็นชื่อจริง: แก้ไฟล์
  `server/data/scores.json` ที่ field `name` ของแต่ละคน (แก้ตอน container
  หยุดทำงาน หรือทำ endpoint เพิ่มเติมได้)
- อยากได้คะแนนต่อ 1 ครั้งไม่เท่ากับ 30: แก้ค่า `FLOWER_POINTS` ใน
  `server/server.js`
- อยากให้ปุ่มกรอกคะแนนต้องใส่รหัสผ่านด้วย: เพิ่ม middleware `requireAdmin`
  ให้กับ route `/api/vote` ใน `server/server.js`

## หมายเหตุด้านความปลอดภัย

หน้าแอดมินใช้รหัสผ่านแบบง่าย (ส่งผ่าน HTTP header) เหมาะสำหรับใช้งานภายใน
เครือข่ายท้องถิ่น (LAN) ของงานเท่านั้น ถ้าจะเปิดให้เข้าถึงจากอินเทอร์เน็ต
ควรเพิ่ม HTTPS และระบบยืนยันตัวตนที่รัดกุมกว่านี้
